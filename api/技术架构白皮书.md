## 执行摘要

本文档描述采购谈判风格测试系统的 API 化技术架构。系统提供 5 个核心 API，支持个人评估、团队分析、风格匹配和课程推荐功能，采用微服务架构设计，确保高可用、高扩展和合规安全。

**核心指标:**
- API 可用性：99.9%
- 响应时间：P95 < 500ms
- 并发能力：1000+ QPS
- 数据安全：等保三级、GDPR 合规

---

## 一、架构愿景

### 1.1 业务目标

```
短期（0-6 个月）:
  - MVP 上线，验证商业模式
  - 支持 1000+ 日活用户
  - 月收入突破¥100,000

中期（6-18 个月）:
  - 企业客户接入
  - API 调用量 100 万+/月
  - 通过等保三级认证

长期（18 个月+）:
  - 开放平台生态
  - 国际化扩展
  - AI 增强评估
```

### 1.2 技术原则

1. **简单优先:** MVP 阶段避免过度设计
2. **云原生:** 容器化、弹性伸缩
3. **安全内置:** 安全不是附加功能
4. **数据驱动:** 所有决策基于数据
5. **渐进演进:** 架构随业务成长

### 3.2 请求处理流程

```
1. 客户端请求
       │
       ▼
2. SLB 负载均衡
       │
       ▼
3. WAF 安全检查
       │
       ▼
4. Kong 网关
   - API Key 验证
   - 限流检查
   - 计费预扣
       │
       ▼
5. 应用服务
   - 参数验证
   - 业务逻辑
   - 数据持久化
       │
       ▼
6. 响应返回
   - 结果数据
   - 计费确认
   - 审计日志
```

## 四、数据架构

### 4.1 数据库设计

**核心实体关系:**
```
┌─────────┐     ┌──────────────┐     ┌──────────┐
│  User   │────<│ Assessment   │>────│  Answer  │
└─────────┘     └──────────────┘     └──────────┘
     │                │
     │                │
     ▼                ▼
┌─────────┐     ┌──────────────┐
│Transaction│    │   Report     │
└─────────┘     └──────────────┘
```

**表结构:**
```sql
-- 用户表
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  phone VARCHAR(20),
  password_hash VARCHAR(255) NOT NULL,
  api_key_hash VARCHAR(255),
  tier VARCHAR(20) DEFAULT 'free',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  deleted_at TIMESTAMPTZ
);

-- 评估表
CREATE TABLE assessments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  type VARCHAR(20) NOT NULL, -- 'full' or 'quick'
  answers JSONB NOT NULL,
  result JSONB,
  status VARCHAR(20) DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 交易表
CREATE TABLE transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  amount DECIMAL(10,2) NOT NULL,
  currency VARCHAR(3) DEFAULT 'CNY',
  type VARCHAR(50) NOT NULL,
  status VARCHAR(20) DEFAULT 'pending',
  payment_method VARCHAR(50),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_assessments_user ON assessments(user_id, created_at);
CREATE INDEX idx_transactions_user ON transactions(user_id, created_at);
```

### 4.2 缓存策略

**缓存层次:**
```
L1: 应用内存缓存 (Node.js Map)
    - 热点配置
    - 会话数据
    TTL: 5 分钟

L2: Redis 缓存
    - 用户资料
    - 评估模板
    - 限流计数
    TTL: 30 分钟 - 24 小时

L3: 数据库
    - 持久化存储
```

**缓存更新策略:**
```typescript
// Cache-Aside 模式
async getUserProfile(userId: string) {
  // 1. 查缓存
  const cached = await redis.get(`user:${userId}`);
  if (cached) return JSON.parse(cached);
  
  // 2. 查数据库
  const user = await db.users.findUnique({ where: { id: userId } });
  
  // 3. 写缓存
  await redis.setex(`user:${userId}`, 1800, JSON.stringify(user));
  
  return user;
}
```

## 五、安全架构

### 5.1 认证授权

**认证流程:**
```
1. 客户端携带 API Key
2. Kong 验证 Key 有效性
3. 查询 Redis 缓存的 Key 信息
4. 注入用户上下文到请求头
5. 应用服务基于上下文授权
```

**权限模型 (RBAC):**
```typescript
enum Role {
  ADMIN = 'admin',
  OPERATOR = 'operator',
  USER = 'user',
  GUEST = 'guest'
}

enum Permission {
  ASSESSMENT_READ = 'assessment:read',
  ASSESSMENT_CREATE = 'assessment:create',
  TEAM_ANALYZE = 'team:analyze',
  PAYMENT_VIEW = 'payment:view'
}
```

### 5.2 数据安全

**加密方案:**
```
传输层：TLS 1.3
存储层：AES-256-GCM
密码：bcrypt (cost=12)
密钥管理：阿里云 KMS
```

**数据分类:**
| 级别 | 数据类型 | 保护措施 |
|------|----------|----------|
| L1 公开 | 课程信息 | 无特殊保护 |
| L2 内部 | 用户资料 | 访问控制 |
| L3 敏感 | 评估结果 | 加密存储 |
| L4 机密 | 支付信息 | 加密 + 审计 |

| 生产 | 线上服务 | 阿里云 ECS 4 核 8G × 3 | ¥1500 |

### 6.2 高可用设计

**多可用区部署:**
```
可用区 A:
  - API 服务 × 2
  - PostgreSQL 主
  
可用区 B:
  - API 服务 × 1
  - PostgreSQL 从
  - Redis 从
  
SLB 跨可用区负载均衡
```

**故障转移:**
```
数据库故障：自动切换到从库（RTO < 60s）
应用故障：自动重启 + 流量切换（RTO < 30s）
区域故障：DNS 切换到备用区域（RTO < 300s）
```

### 6.3 弹性伸缩

**自动伸缩规则:**
```yaml
# Kubernetes HPA
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: api-service-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: api-service
  minReplicas: 2
  maxReplicas: 20
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```
