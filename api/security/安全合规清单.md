## 一、数据加密方案

### 1.1 传输加密

**要求:** 所有数据传输必须加密

**实施方案:**
```yaml
TLS 版本：TLS 1.3（最低 TLS 1.2）
证书：Let's Encrypt 或商业证书
HSTS: max-age=31536000; includeSubDomains
证书固定：移动端 APP 启用
```

**配置示例（Nginx/Kong）:**
```nginx
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
ssl_prefer_server_ciphers on;
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```

**检查清单:**
- [ ] HTTPS 强制跳转
- [ ] 无效证书拦截
- [ ] TLS 版本检测
- [ ] 弱加密套件禁用

### 1.3 数据脱敏

**要求:** 日志和展示时脱敏敏感信息

**脱敏规则:**
```javascript
// 手机号：138****1234
function maskPhone(phone) {
  return phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2');
}

// 邮箱：u**r@example.com
function maskEmail(email) {
  return email.replace(/^(.).*(.)@/, '$1**$2@');
}

// 身份证：110101********1234
function maskIDCard(id) {
  return id.replace(/(\d{6})\d{8}(\d{4})/, '$1********$2');
}
```

**日志脱敏中间件:**
```typescript
// NestJS 中间件示例
@Injectable()
export class LoggingMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: NextFunction) {
    const safeBody = this.sanitize(req.body);
    this.logger.log('Request', { body: safeBody });
    next();
  }
  
  private sanitize(data: any): any {
    // 移除或脱敏敏感字段
    const sensitive = ['password', 'api_key', 'id_card', 'phone'];
    // ... 实现脱敏逻辑
  }
}
```

**检查清单:**
- [ ] 日志中无明文密码
- [ ] 日志中无完整手机号
- [ ] 日志中无完整身份证
- [ ] 错误信息不泄露敏感数据

### 2.4 数据泄露响应

**要求:** 72 小时内报告

**应急预案:**
```
1. 发现泄露 → 立即隔离
2. 评估影响 → 确定范围
3. 通知监管 → 72 小时内
4. 通知用户 → 高风险时
5. 修复漏洞 → 优先处理
6. 事后复盘 → 改进措施
```

**通知模板:**
```
尊敬的用户：

我们发现 [日期] 发生了数据安全事件，可能影响了您的 [数据类型]。

受影响范围：[具体说明]
我们已采取的措施：[说明]
建议您采取的措施：[建议]
联系方式：[support@...]

[公司名称]
[日期]
```
