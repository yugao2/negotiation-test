## 一、API 网关选型

### 对比矩阵

| 维度 | Kong | Apigee | AWS API Gateway |
|------|------|--------|-----------------|
| **部署方式** | 自托管/云 | SaaS | SaaS |
| **月成本** | ¥500-2000 | ¥8000+ | ¥1000-5000 |
| **性能** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **插件生态** | 丰富 | 丰富 | 中等 |
| **学习曲线** | 中 | 高 | 低 |
| **国内支持** | 好 | 一般 | 好 |
| **定制能力** | 高 | 中 | 低 |

### 详细分析

#### Kong（推荐 ✅）
**优势:**
- 开源核心，可自托管控制成本
- 插件丰富（限流、认证、日志、计费）
- 性能优异（基于 Nginx/OpenResty）
- 国内有技术支持团队
- 适合 MVP 快速迭代

**劣势:**
- 需要自行运维
- 企业版功能需付费

**成本估算:**
```
初期（自托管）: ¥500/月（服务器）
成长期（Kong Cloud）: ¥2000/月
```

#### Apigee
**优势:**
- Google 背书，企业级功能完整
- 分析能力强
- 安全性高

**劣势:**
- 成本高（¥8000+/月）
- 国内访问速度一般
- 锁定风险

**结论:** 不适合初创项目

#### AWS API Gateway
**优势:**
- 与 AWS 生态无缝集成
- 按量付费，初期成本低
- 运维简单

**劣势:**
- 冷启动延迟
- 复杂定制困难
- 长期成本不可控

**结论:** 备选方案

## 二、数据库选型

### PostgreSQL vs MongoDB

| 维度 | PostgreSQL | MongoDB |
|------|------------|---------|
| **数据模型** | 关系型 | 文档型 |
| **事务支持** | ACID 完整 | 4.0+ 支持 |
| **查询能力** | SQL 强大 | 灵活但复杂 |
| **扩展性** | 垂直为主 | 水平分片 |
| **学习成本** | 中 | 低 |
| **生态成熟度** | 30 年 + | 15 年 + |

### 项目需求分析

**数据结构:**
- 用户信息（结构化）→ PostgreSQL
- 评估答案（半结构化）→ 两者皆可
- 分析报告（文档）→ MongoDB 更适合
- 交易记录（强一致性）→ PostgreSQL

### 推荐方案：PostgreSQL + JSONB（✅）

**理由:**
1. 评估答案可用 JSONB 存储，兼顾灵活性与查询能力
2. 交易记录需要 ACID 保证
3. 团队更熟悉 SQL
4. 可后期用 TimescaleDB 做分析

**表结构设计:**
```sql
-- 用户表
CREATE TABLE users (
  id UUID PRIMARY KEY,
  email VARCHAR(255) UNIQUE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 评估记录
CREATE TABLE assessments (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  answers JSONB NOT NULL,
  result JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 交易记录
CREATE TABLE transactions (
  id UUID PRIMARY KEY,
  user_id UUID,
  amount DECIMAL(10,2),
  status VARCHAR(50),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**成本:**
- 阿里云 RDS PostgreSQL: ¥300/月（基础版）
- 自建（Docker）: ¥0（开发环境）

## 三、缓存选型

### Redis vs Memcached

| 维度 | Redis | Memcached |
|------|-------|-----------|
| **数据结构** | 丰富（String/List/Hash/Set/ZSet） | 仅 String |
| **持久化** | 支持（RDB/AOF） | 不支持 |
| **集群** | 原生支持 | 需客户端分片 |
| **发布订阅** | 支持 | 不支持 |
| **Lua 脚本** | 支持 | 不支持 |
| **性能** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

### 推荐：Redis（✅）

**使用场景:**
1. **会话缓存:** 用户登录态（2 小时 TTL）
2. **API 限流:** 滑动窗口计数
3. **热点数据:** 评估模板、课程列表
4. **分布式锁:** 防止重复扣费
5. **消息队列:** 异步处理报告生成

**配置建议:**
```yaml
# redis.conf
maxmemory: 2gb
maxmemory-policy: allkeys-lru
appendonly: yes
appendfsync: everysec

# 集群模式（成长期）
nodes: 3 (1 master + 2 replicas)
```

**成本:**
- 阿里云 Redis: ¥200/月（2GB 主从）
- 自建：¥0（开发环境）

## 四、部署方案

### 阶段一：MVP（0-3 个月）

**架构:**
```
┌─────────────┐
│   Kong      │  (API 网关，Docker)
├─────────────┤
│   App       │  (Node.js/Python, Docker)
├─────────────┤
│ PostgreSQL  │  (Docker)
│    Redis    │  (Docker)
└─────────────┘
     │
     ▼
┌─────────────┐
│  阿里云 ECS  │  (2 核 4G, ¥200/月)
└─────────────┘
```

**工具链:**
- Docker Compose（本地开发）
- GitHub Actions（CI/CD）
- 阿里云镜像仓库

**成本:** ¥500/月

---

### 阶段二：成长期（3-12 个月）

**架构:**
```
┌─────────────────────────┐
│   阿里云 SLB (负载均衡)  │
└───────────┬─────────────┘
            │
    ┌───────┼───────┐
    ▼       ▼       ▼
┌───────┐ ┌───────┐ ┌───────┐
│ Kong  │ │ Kong  │ │ Kong  │
│ App   │ │ App   │ │ App   │
└───────┘ └───────┘ └───────┘
    │         │         │
    └─────────┼─────────┘
              ▼
    ┌─────────────────┐
    │  RDS PostgreSQL │
    │  Redis 集群     │
    └─────────────────┘
```

**工具链:**
- Kubernetes（ACK）
- Helm Charts
- Prometheus + Grafana（监控）

**成本:** ¥3000-5000/月

---

### 阶段三：成熟期（12 个月+）

**架构:**
- 多可用区部署
- 读写分离
- CDN 加速静态资源
- 异地灾备

**成本:** ¥10000+/月

## 五、技术栈总览

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| **API 网关** | Kong | 开源、灵活、成本低 |
| **应用框架** | Node.js (NestJS) | 类型安全、生态好 |
| **数据库** | PostgreSQL 15 | ACID、JSONB 灵活 |
| **缓存** | Redis 7 | 多功能、高性能 |
| **消息队列** | Redis Streams | 轻量、无需额外组件 |
| **容器** | Docker + K8s | 标准化、可扩展 |
| **CI/CD** | GitHub Actions | 免费、集成好 |
| **监控** | Prometheus + Grafana | 开源、强大 |
| **日志** | ELK Stack | 成熟、可搜索 |
| **云厂商** | 阿里云 | 国内访问快、支持好 |

## 六、风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |

## 七、决策总结

**最终选型:**
```
Kong + NestJS + PostgreSQL + Redis + 阿里云
```

**核心理由:**
1. 成本低（初期¥500/月）
2. 技术成熟，招聘容易
3. 可扩展，支持平滑升级
4. 国内访问速度快
5. 社区活跃，问题易解决
