# 发明专利申请书

## 发明名称
**采购风格与供应商匹配度评估方法**

---

## 一、技术领域

本发明属于供应链管理与供应商关系管理技术领域，具体涉及一种采购风格与供应商匹配度评估方法，尤其适用于采购人员与供应商类型的最优配对。

---

## 二、背景技术

### 2.1 现有技术现状

当前供应商管理中的人员配置主要存在以下问题:

1. **随机分配**: 采购人员与供应商的配对多基于随机分配或管理者主观判断
2. **忽视风格匹配**: 未考虑采购人员风格与供应商类型的匹配度
3. **缺乏评估工具**: 缺少科学的匹配度评估工具
4. **效果难以追踪**: 配对效果缺乏量化评估与反馈机制
5. **经验依赖**: 优秀配对多基于个人经验，难以复制推广

### 2.2 现有技术缺陷

| 现有技术 | 缺陷 |
|---------|------|
| 供应商分类管理 | 仅分类供应商，无人员匹配 |
| 采购轮岗制度 | 定期轮换，忽视匹配连续性 |
| 供应商绩效考核 | 侧重结果，忽视人员因素 |
| 客户关系管理 (CRM) | 侧重销售端，采购端应用少 |

### 2.3 本发明要解决的技术问题

1. 提供采购人员风格与供应商类型的匹配评估方法
2. 建立匹配度量化评估模型
3. 输出最优人员 - 供应商配对建议
4. 支持匹配效果追踪与优化

---

## 三、发明内容

### 3.1 技术方案概述

本发明提供一种采购风格与供应商匹配度评估方法，包括以下步骤:

**步骤 S1**: 采购人员风格测评
- 采用 16 种动物形象人格模型
- 输出采购人员风格画像
- 记录历史谈判表现数据

**步骤 S2**: 供应商类型分类
- 基于供应商特征进行分类
- 定义供应商合作风格
- 记录历史合作数据

**步骤 S3**: 匹配度评估模型构建
- 建立人员风格 - 供应商类型匹配矩阵
- 定义匹配度评分规则
- 训练匹配效果预测模型

**步骤 S4**: 匹配度计算与配对建议
- 计算人员 - 供应商匹配度得分
- 生成最优配对建议
- 输出匹配风险评估

**步骤 S5**: 匹配效果追踪与优化
- 收集合作效果反馈数据
- 验证匹配度预测准确性
- 持续优化匹配模型

### 3.2 供应商类型分类模型

**分类维度 1: 合作导向**
| 类型 | 特征 | 合作风格 |
|-----|------|---------|
| 战略型 | 长期合作、深度绑定 | 共赢导向、信息共享 |
| 交易型 | 短期交易、价格敏感 | 竞争导向、信息保密 |
| 伙伴型 | 中期合作、互信基础 | 协作导向、适度共享 |
| 机会型 | 临时合作、灵活多变 | 灵活导向、按需合作 |

**分类维度 2: 沟通风格**
| 类型 | 特征 | 沟通偏好 |
|-----|------|---------|
| 正式型 | 流程规范、文档齐全 | 正式沟通、书面确认 |
| 灵活型 | 流程灵活、口头约定 | 非正式沟通、快速响应 |
| 分析型 | 数据驱动、细节关注 | 数据沟通、精准表达 |
| 关系型 | 关系优先、人情导向 | 关系沟通、情感连接 |

**分类维度 3: 决策风格**
| 类型 | 特征 | 决策特点 |
|-----|------|---------|
| 集权型 | 高层决策、集中管控 | 决策慢、执行快 |
| 分权型 | 授权决策、分散管理 | 决策快、执行快 |
| 共识型 | 集体决策、多方参与 | 决策慢、执行中 |
| 专业型 | 专家决策、技术导向 | 决策中、执行准 |

### 3.3 人员 - 供应商匹配矩阵

**匹配矩阵示例**:

| 采购人员风格 | 战略型供应商 | 交易型供应商 | 伙伴型供应商 | 机会型供应商 |
|-------------|-------------|-------------|-------------|-------------|
| 🦁 狮子 (主导) | 中 (70%) | 高 (90%) | 中 (65%) | 高 (85%) |
| 🦅 鹰 (战略) | 高 (95%) | 低 (40%) | 高 (80%) | 中 (60%) |
| 🦊 狐狸 (灵活) | 中 (75%) | 高 (85%) | 高 (80%) | 高 (95%) |
| 🦉 猫头鹰 (分析) | 高 (85%) | 中 (70%) | 高 (80%) | 低 (50%) |
| 🐬 海豚 (关系) | 高 (90%) | 低 (45%) | 高 (95%) | 中 (65%) |
| 🐘 大象 (稳健) | 高 (85%) | 中 (65%) | 高 (90%) | 低 (45%) |
| 🐯 老虎 (强势) | 中 (60%) | 高 (85%) | 中 (65%) | 高 (80%) |
| 🐺 狼 (团队) | 高 (85%) | 低 (50%) | 高 (90%) | 中 (60%) |

**匹配度评分规则**:
```
匹配度 = w1×风格兼容性 + w2×沟通匹配度 + w3×决策协同度 + w4×历史效果分
权重: w1=0.3, w2=0.3, w3=0.2, w4=0.2
匹配度 ∈ [0, 100%]
```

### 3.4 匹配度评估指标

**指标 1: 风格兼容性 (30%)**
- 评估采购人员风格与供应商合作风格的兼容程度
- 基于匹配矩阵查表得分
- 考虑风格互补性

**指标 2: 沟通匹配度 (30%)**
- 评估双方沟通偏好的匹配程度
- 正式型 vs 正式型 = 高匹配
- 关系型 vs 关系型 = 高匹配
- 正式型 vs 灵活型 = 低匹配

**指标 3: 决策协同度 (20%)**
- 评估双方决策风格的协同程度
- 集权型 vs 集权型 = 高协同
- 分权型 vs 分权型 = 高协同
- 集权型 vs 共识型 = 低协同

**指标 4: 历史效果分 (20%)**
- 基于历史合作效果数据
- 合作成功率、谈判达成率、满意度评分
- 有历史数据时权重提升

### 3.5 匹配效果预测模型

**机器学习模型**:
```
输入特征:
- 采购人员风格向量 (16 维)
- 供应商类型向量 (12 维)
- 历史合作数据 (如有)
- 采购场景特征 (5 维)

输出:
- 匹配度得分 (0-100%)
- 合作成功率预测
- 风险评估

模型类型:
- 梯度提升树 (GBDT)
- 随机森林
- 神经网络 (数据量充足时)
```

### 3.6 配对建议生成

**高匹配度 (>80%)**: 
- 推荐配对
- 优先安排合作
- 建立长期关系

**中匹配度 (60-80%)**:
- 可接受配对
- 需要关注沟通
- 定期评估效果

**低匹配度 (<60%)**:
- 谨慎配对
- 需要第三方协调
- 考虑人员调整

## 五、具体实施方式

### 实施例 1: 战略供应商人员配置

**场景**: 某企业有 5 家战略供应商，需要配置最优采购人员

**步骤**:
1. 对 10 名采购人员进行风格测评
2. 对 5 家战略供应商进行分类:
   - 供应商 A: 战略型 + 正式型 + 集权型
   - 供应商 B: 战略型 + 分析型 + 共识型
   - 供应商 C: 伙伴型 + 关系型 + 分权型
   - 供应商 D: 战略型 + 正式型 + 专业型
   - 供应商 E: 伙伴型 + 灵活型 + 分权型
3. 计算匹配度矩阵
4. 生成最优配对:
   - 供应商 A → 鹰型人员 (匹配度 92%)
   - 供应商 B → 猫头鹰型人员 (匹配度 88%)
   - 供应商 C → 海豚型人员 (匹配度 95%)
   - 供应商 D → 大象型人员 (匹配度 85%)
   - 供应商 E → 狐狸型人员 (匹配度 90%)
5. 实施配对并跟踪效果

### 实施例 2: 供应商切换人员调整

**场景**: 供应商更换对接人员，需要重新匹配

**步骤**:
1. 评估原人员 - 供应商匹配度与效果
2. 对新人员进行风格测评
3. 计算新匹配度
4. 如匹配度下降>15%，提出预警
5. 提供过渡期支持方案

### 实施例 3: 新供应商人员分配

**场景**: 引入新供应商，需要分配负责人员

**步骤**:
1. 对新供应商进行分类评估
2. 从现有人员中筛选匹配候选人
3. 计算匹配度并排序
4. 选择最优匹配人员
5. 建立初期合作跟踪机制

## 六、系统实现

### 6.1 数据库设计

**人员风格表**:
```sql
CREATE TABLE buyer_style (
    buyer_id VARCHAR(32) PRIMARY KEY,
    buyer_name VARCHAR(64),
    primary_type VARCHAR(32),  -- 主类型 (如：鹰)
    secondary_type VARCHAR(32), -- 副类型
    style_vector JSON,          -- 16 维风格向量
    created_time DATETIME
);
```

**供应商类型表**:
```sql
CREATE TABLE supplier_type (
    supplier_id VARCHAR(32) PRIMARY KEY,
    supplier_name VARCHAR(128),
    cooperation_type VARCHAR(32),  -- 合作导向类型
    communication_type VARCHAR(32), -- 沟通风格类型
    decision_type VARCHAR(32),      -- 决策风格类型
    type_vector JSON,               -- 类型向量
    created_time DATETIME
);
```

**匹配记录表**:
```sql
CREATE TABLE match_record (
    record_id VARCHAR(32) PRIMARY KEY,
    buyer_id VARCHAR(32),
    supplier_id VARCHAR(32),
    match_score DECIMAL(5,2),  -- 匹配度得分
    cooperation_type VARCHAR(32),
    start_date DATE,
    status VARCHAR(16),
    effect_feedback JSON,       -- 效果反馈
    created_time DATETIME
);
```

### 6.2 匹配度计算引擎

```python
def calculate_match_score(buyer, supplier, history=None):
    # 风格兼容性
    style_compat = lookup_match_matrix(buyer.primary_type, supplier.cooperation_type)
    
    # 沟通匹配度
    comm_match = evaluate_communication_match(buyer, supplier)
    
    # 决策协同度
    decision_sync = evaluate_decision_sync(buyer, supplier)
    
    # 历史效果分
    history_score = 0
    if history:
        history_score = calculate_history_score(history)
    
    # 加权计算
    match_score = (
        0.3 * style_compat +
        0.3 * comm_match +
        0.2 * decision_sync +
        0.2 * history_score
    )
    
    return match_score
```
